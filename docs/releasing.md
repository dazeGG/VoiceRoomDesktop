# Releasing Voice Room Desktop

Releases are built by GitHub Actions from git tags. Do not upload desktop artifacts manually.

## Versioning

Use semver in `package.json` and tag releases as `vX.Y.Z`.

The release workflow validates that the tag matches `package.json`:

```text
package.json version: 1.0.2
git tag: v1.0.2
```

If they do not match, CI fails before building installers.

## Create a release

1. Update the app version:

   ```bash
   npm version 1.0.2 --no-git-tag-version
   ```

2. Commit the version bump and any release changes:

   ```bash
   git add package.json package-lock.json
   git commit -m "Release v1.0.2"
   ```

3. Create and push the tag:

   ```bash
   git tag v1.0.2
   git push origin main
   git push origin v1.0.2
   ```

4. GitHub Actions builds:

   - macOS `.dmg` on a macOS runner.
   - Windows `.exe` on a Windows runner with MSVC.
   - Native audio helpers on their matching platforms.

5. The workflow creates a GitHub Release and uploads the artifacts.

Release artifacts must include the update metadata generated by `electron-builder`:

- Windows: `latest.yml`, `.exe`, `.blockmap`
- macOS: `latest-mac.yml`, `.zip`, `.dmg`, `.blockmap`

Packaged desktop builds use these files for the startup update gate. If update metadata or artifacts cannot be reached at launch, the app should still allow entry when the configured Voice Room site is reachable, while clearly reporting the update failure. If the site is unreachable too, the launch screen stays blocked with a site-unavailable error.

## Configuration

The workflow uses the repository variable `VOICE_ROOM_URL` when it is set. If the variable is absent, it falls back to:

```text
https://voiceroom.ru
```

Set it in GitHub:

```text
Settings -> Secrets and variables -> Actions -> Variables -> VOICE_ROOM_URL
```

## Signing

Windows portable `.exe` files are signed through SignPath after the unsigned CI build completes. See [code-signing.md](code-signing.md).

macOS artifacts are currently unsigned. Future production releases may add Apple Developer ID signing and notarization.
